UX vs Intent Paradigm
The most fundamental shift in AI: from clicking through interfaces to expressing intent in natural language.

Traditional UX OLD

  • User → Interface → Action
  • User must know the steps
  • Fixed navigation paths
  • 14+ clicks to book a hotel
  • User drives every micro-step

Agentic UX NEW

  • User Intent → Agent → Actions
  • User just states the goal
  • Agent plans the path
  • 3 conversational messages
  • Agent drives execution

Core Concept
Autonomous Orchestration
The agent doesn't just respond — it plans a sequence of tool calls, handles errors, loops when needed, and delivers a final result. The user only sees intent in and outcome out.
Five Powers of AI Agents
Every AI agent is built from these five foundational capabilities. Together, they enable full autonomous task execution.
👁️
See
Visual Understanding — images, screenshots, documents, UI states
👂
Hear
Audio Processing — voice commands, transcription, tone recognition
🧠
Reason
Decision Making — planning, analysis, choosing the right tool or action
⚡
Act
Execution — calling APIs, running code, writing files, using tools
💾
Remember
Memory & Context — conversation history, user preferences, past results

Key Insight
Reason is the central orchestrator — it reads outputs from See/Hear/Remember and decides what to Act on next. The loop continues until the task is complete.

Model Context Protocol (MCP)
The universal connector for AI agents. Imagine USB — one standard port that works with any device. MCP does the same for AI tools.
The M×N Problem MCP Solves

Before MCP — M×N Chaos

  • 5 models × 10 tools = 50 integrations
  • Each pair needs custom code
  • Breaking changes everywhere
  • Maintenance nightmare

After MCP — M+N Clean

  • 5 models + 10 tools = 15 connectors
  • One standard protocol
  • Any model ↔ any tool
  • Add once, use everywhere
What MCP Exposes
🛠️
Tools
Functions the agent can call — search, write file, send email, query DB
📄
Resources
Data the agent can read — files, database rows, API responses
💬
Prompts
Pre-built instruction templates shared by the server
Bidirectional Communication — Sampling
Unlike simple APIs, MCP supports sampling: the server can ask the model to generate content mid-task. It's a two-way channel, not just client-to-server.
Evolution of AI
Three distinct eras of AI capability — each one enabled by breakthroughs in the previous era.
📊
Predictive AI
Learns patterns from data to forecast outcomes. Spam filters, recommendation engines, price prediction. Narrow but highly accurate in its domain.
✍️
Generative AI
Creates new content — text, images, code, audio. GPT-4, Midjourney, Claude. Can answer questions, write essays, generate code from natural language.
🤖
Agentic AI
Acts autonomously to complete multi-step tasks. Uses tools, loops, plans. Doesn't just respond — it executes. This is where we are in 2025-2026.
Skill Era vs Prompt Era

Prompt Era Old

Skill Era Now

Skills, AGENTS.md & goose
The practical layer of the AI stack — how expertise is packaged, behavior is governed, and production agents are structured.
AGENTS.md — Behavior Rules File
Build Rules
How to run the project, install deps, compile, test. Agent reads this before coding.
Style Rules
Code formatting, naming conventions, file structure. Keeps output consistent.
Security Rules
What NOT to do. Never expose API keys, never delete prod data, never skip auth.
Hierarchy
Global → Project → Directory → File. More specific rules override broader ones.
SKILL.md — Expertise Packaging
How a SKILL.md works
1. Agent encounters a task (e.g. "create a Word doc")
2. Checks if a relevant SKILL.md exists
3. Reads the skill file for best practices
4. Executes the task using the skill's guidance
5. Output is professional-grade, not generic AI output
goose — Reference Agent
Open-source production blueprint
goose is Block's open-source agent that implements the full AIFF stack — MCP, AGENTS.md, Skills, and production patterns. Study it to understand real-world agent architecture without building from scratch.
Big Concepts & Mental Models
The conceptual frameworks that tie everything together. These are the ideas you'll keep returning to.
🔌
USB Analogy — Standardization
Before USB: every device had its own port. After USB: one standard, everything works. MCP is USB for AI — one protocol, any model connects to any tool.
🆚
Skills vs MCP — Knowledge vs Connection
MCP is about connecting agents to tools (infrastructure). Skills are about knowing how to use those tools well (expertise). You need both.
✈️
Portable AI Systems
Your agent, its skills, and its tools should move freely between Claude, GPT, Gemini — and between Cursor, VS Code, goose. No platform should own your stack.
👤
Digital FTE Concept
An agent with the right Five Powers, connected via MCP to relevant tools, guided by AGENTS.md rules and loaded with domain SKILL.md files = a Digital Full-Time Employee. A virtual worker that never sleeps.
